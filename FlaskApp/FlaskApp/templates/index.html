Some Of My Projects

Below are a few projects which I am proud to have been a part of.

EyeSpy - January 2018
During the Spring 2018 semester, a few of my friends and I went to a hackathon at UPenn, PennApps, and created a two wheeled robot which was controlled by a user's eye movement. Using Tensorflow, we trained a neural network to classify the user's eyeball, and pupil. From here, we were able to determine the direction the user was looking and made the robot turn in that direction.
Twitter Stock Bot - February 2019
At the 2019 Stony Brook CEWIT Hackathon, I had the idea of attempting to correlate Stock Prices with Social Media influence. Using this concept, my team and I developed an application which pulls relevant articles about select Stocks, maps the content to a positive or negative rating, then tries to predict how the stock will perform over the course of the day.
Quadcopter - January 2018
Over Freshman Winter break, eager to work on a project, I began working on a Quadcopter as a personal project along with two of my close friends. Our setup consists of using two arduinos to communicate between eachother through radio frequency where one functions as a controller and the other as the drone. The frame of the drone is a carbon fiber plate which holds four brushless motors, an ultrasonic sensor, and a gyroscope for self-leveling capabilities. Since this was my first real project, it taught me a significant amount about the software development process for any project.

Work Experience:
During my freshman summer, I was fortunate enough to work as an intern at Zebra Technologies.
Throughout the summer, I worked alongside three other talented individuals developing three different applications, each following the agile methodology.
  1. First Aid Application:
    • The objective for this application was to rework a previously developed android application, adding functionality and creating an intuitive and user interface.
    • The application we created allowed the buildings First Responders to communicate using Push To Talk, as well as keep track of the items within their first aid bags, and the first aid cabinets around the building.
    • This application is currently being utilized in three Zebra Technologies locations and there are plans to get this application into other locations.
    • My role in the development of this application consisted of implementing new features, fixing previous bugs, and developing database queries to integrate the application with the AWS server the inventory of each medicine bag and cabinet.
  2. Zebra Event Registration Kiosk:
    • The second project our team was given was to create a portal to enable Zebra employees to register for different events hosted by Zebra.
    • Our solution to this assignment was to create a web application which served as an interface for the backend MySQL server hosted on AWS. This web application enabled the administrators to perform actions such as creating an event, adding images to events, see the list of currently enrolled employees, send emails to the regsitrants, etc. More importantly, all events were not collected into one single database.
    • We then created an android application which pulled the event data from the AWS server, displayed the upcoming events, and allowed employees to register for the events. Prioritizing speed, we wanted to allow employees to register by simply scanning the barcode on their badge, or taking an image of the name on the badge. Using OCR, and the built-in scanner, we were able to gather the user's information from the Zebra Employee Database and register employees for the event in seconds.
    • For this project, my contributions ranged from created the android application, to implementing the OCR and Scanning features, to setting up the administrative web application and database.
  3. Innovation Tracker:
    • The third and final project our team was assigned was to reinvent the Innovation Tracker.
    • To preface this, the Innovation Tracker was a simply excel form which was submitted by each Zebra engineer monthly in order to record their engineering activities.
    • Each engineer was expected to complete a minimum of 8 points per year, where each activity they completed ranged from 1-4 points. After the month ended, the system would generate reports for their managers.
    • Our team was tasked with producing a solution to this application which would be more simple and intuitive.
    • After discussing with various managers, we opted for a web application where an employee could enter their points, managers could view their team's points, and we could have one centralized location for getting these statistics.
    • Our application is currently in use by hundreds of Zebra engineers, being used to perform yearly reviews, and encourage professional development within all Zebra locations.

After the summer, I was given the offer to continue my time at Zebra, working part time during the semester and full time again the following summer.
