Computer Science III is a course dedicated to reinforcing concepts and techniques learned through
previous CS courses such as Computer Science II.
Students will obtain practical programming skills and experience through the rigorous assignments
  • Using JavaFX, this course was based around creating a Data & Algorithm Visualizer.
  • Each of the homework assignments focused on taking steps towards creating the final application which ended up
    being between 10,000 and 20,000 lines of code.
    The homework assignments were based off the following topics:
    • Setting up basic graphics and getting introduced to JavaFX.
    • Programmatically reacting to user induced events, and making the application react accordingly.
    • Learning how to debug applications to find memory leaks, time consuming functions, and how to optimize software to reduce these problems after performing Unit Tests.
    • Creating various UML diagrams for each of the classes as well as diagrams representing each of the Use Cases.
      Then compacting each of the diagrams into a 36 page application Software Design Document.
    • Implementing all the required functionality using Agile methodology to create a functioning and aesthetically pleasing application.
  • This course involved learning how to use Git to perform version control which was used throughout the creation of the course-long project.
  • The application we created also utilized multi-threading capabilities to allow the User Interface
    to remain responsive while performing expensive calculations on other threads as we studied synchronization, locks, and concurrency.
  • After threading, we then proceeded to study the importance of
    • UML diagrams & Object Representation in UML
    • Use Case Diagrams
    • Interfaces
    • Subclasses & Inheritances
    • Generic Types & Bounded Generics
    • Hash Codes & Equality
    • Cloning(Shallow & Deep Cloning)
    • Serialization of Objects
  • Computer Science III also went in depth about design patterns and how they can positively influence your software.
    The following design patterns were discussed
    • Iterator Pattern
    • Command Pattern
    • Observer Pattern
    • Strategy Pattern
    • Composite Pattern
    • Decorator Pattern
    • Factory Pattern
    • Singleton Pattern
    • Adapter Pattern
    • Builder Pattern
    • Prototype Pattern
    • Facade Pattern
  • Finally, the course ended with the discussion of Profiling, Optimization, Debugging, and Testing.
